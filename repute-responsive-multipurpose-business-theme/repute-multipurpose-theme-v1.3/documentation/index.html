Overview

Repute is a multipurpose website template based on Bootstrap framework. Save your time, create high quality website faster either for business, corporate, agency or even personal. Repute provides you many options and features to fulfill your website requirements. With on-going development, Repute will have additional options and features in every release update, for free. It is easy to customize Repute since it's built with developer-friendly and clean-coded in mind. Even better, we provide you extensive documentation to get started or dig more. If you use Sass and Compass, you have a good news because Repute is built with Sass and Compass. If you have any problem using the theme, do not hesitate to reach our support for free.

Important Notes

Recommended: If you don't use Sass, it's recommended that you write your own styles to file assets/css/my-custom-styles.css. Avoid any direct change to main.css for easier theme update release in the future.
Note: Newsletter and contact form only works in web server with PHP environment. We recommend you to develop the theme in web server, i.e. XAMPP, MAMP or others.
Custom Skin: If you need exact theme color, you can either copy and modify the existing skin file or send your color skin request via email to support@themeineed.com. Provide your License ID and one hex color value (accent color) in the message, then we will reply you with skin file attached.

Working With Newsletter

In footer section of the theme, you will find newsletter form which work with mailchimp. You can find the AJAX call script for newsletter at assets/js/repute-scripts.js and find "newsletter". As you can see, the script calls php/mailchimp.php to do the backend process. That's the file where we put some newsletter parameters setting. Note: In order to make the newsletter form works, please make sure the theme is placed in web server with PHP environment, otherwise it doesn't work (i.e. opening the template as static file instead on localhost).

One Page

One-page HTML is basically derived from main theme, only it has 1-level menu with scrolling navigation and just consists of one page for the whole content. Consequently, the CSS, Javascript, images and others are taken from the same assets/ directory as main theme. Exception is made if only the CSS or Javasript needed is specific to the one-page only, for example at one-page/assets/js/ directory. However, this is a way to simplified things and avoid too many code/asset files duplication. In your one-page project, you may be just copy the asset files necessary from main theme and work in separate directory.

Blog Templates

Blog templates located at blog/ directory don't inherit assets from main theme, including css, image, javascript and so on. Although looks similar, blog templates do not inherit styles from main theme, they have their own styles for each version, such as blog-home-v1.css, blog-home-v2.css.

The functionality itself is not really different from main theme, such as post carousel by Slick plugin. You can find guide about how to use Slick plugin in this documentation, you can learn it from Product Carousel guide. The other is newsletter function which has been explained at Working With Newsletter..

Twitter fetcher is new features provided in this theme, implemented in Blog Home v2 at the sidebar and based on Twitter Post Fetcher plugin. Please see below for the guides.

You can find how the plugin is configured and initialized at blog/assets/js/repute-blog-scripts.js

This configuration uses custom templating and custom date formatter for tweet output. The main concern here is "id" parameter, please follow these steps to get your id:

  1. Go to www.twitter.com and sign in as normal, go to your settings page.
  2. Go to "Widgets" on the left hand side.
  3. Create a new widget for what you need eg "user timeline" or "search" etc.
  4. Feel free to check "exclude replies" if you dont want replies in results.
  5. Now go back to settings page, and then go back to widgets page, you should see the widget you just created. Click edit.
  6. Now look at the URL in your web browser, you will see a long number like this: 441767385733668865
  7. Use this as your ID below instead!
